## 简介

Claude Code 是 Anthropic 推出的官方 CLI 工具，让你能够在终端中与 Claude AI 助手交互。它不仅能回答编程问题，还能直接操作你的代码库——读取文件、编辑代码、运行命令等。

### 主要特点
- 💻 直接在终端使用，无需打开浏览器
- 🛠️ 可以读取、编辑、创建文件
- 🔍 可以搜索代码库（grep、glob）
- 🚀 可以执行 bash 命令
- 🤖 支持代理（Agent）模式执行复杂任务

## 安装与启动

### 安装要求
- macOS 或 Linux（Windows 可通过 WSL 使用）
- Node.js 16+

### 安装方法

```bash
# 使用 npm 安装
npm install -g @anthropic-ai/claude-code

# 或使用 npx（无需全局安装）
npx @anthropic-ai/claude-code
```

### 首次启动

```bash
# 在项目目录中启动
claude

# 或在特定目录启动
claude /path/to/project
```

首次使用需要进行身份验证，按提示登录 Anthropic 账户。

## 基本使用

### 界面说明

启动后会看到类似下面的界面：

```
> claude
╭─────────────────────────────────────────╮
│  Claude Code v0.x.x                     │
╰─────────────────────────────────────────╯

/Users/seven/my-project
>
```

光标处的 `>` 是输入提示符，在这里输入问题或指令。

### 基本对话

直接输入自然语言即可与 Claude 交流：

```
> 帮我解释一下这个项目的结构
> 如何在这个项目中添加一个新的 API 端点？
> 这段代码有什么 bug 吗？
```

### 多行输入

需要输入多行内容时，使用三个反引号包裹：

```
> ```
function calculateSum(a, b) {
    return a + b;
}
```
```

## 核心功能详解

Claude Code 的强大之处在于它能直接操作你的代码库。以下是主要功能：

### 1. 文件操作

#### 读取文件
Claude 可以读取项目中的任何文件：

```
> 帮我看看 src/main.js 文件的内容
> 读取 README.md 文件
```

#### 编辑文件
Claude 可以直接修改代码：

```
> 在 src/main.js 中添加一个 console.log
> 把 config.json 中的 timeout 改成 5000
```

#### 创建文件
```
> 创建一个 tests/utils.test.js 文件，包含基本的测试代码
```

### 2. 代码搜索

#### Glob 搜索（按文件名模式）
查找符合特定模式的文件：

```
> 帮我找到所有的 React 组件文件（**/*.tsx）
> 查找所有的测试文件（**/*.test.js）
```

#### Grep 搜索（按内容搜索）
在代码中搜索特定内容：

```
> 搜索项目中所有用到 "axios" 的地方
> 查找所有包含 "TODO" 的注释
```

### 3. 命令执行

Claude 可以运行 bash 命令：

```
> 运行 npm install 安装依赖
> 执行 npm test 运行测试
> 查看 git status
```

**安全提示**：执行命令前，Claude 会询问你的许可。

### 4. 代理模式（Agent）

对于复杂的多步骤任务，可以使用代理模式：

```
> @agent 帮我重构这个项目的目录结构
> @agent 实现一个完整的用户认证系统
```

代理会：
- 自动规划任务步骤
- 自主执行多个操作
- 处理中间结果
- 完成整个任务

## 特殊命令

### / 命令（斜杠命令）

Claude Code 支持多种特殊命令：

| 命令 | 功能 |
|------|------|
| `/help` | 显示帮助信息 |
| `/clear` | 清空对话历史 |
| `/exit` 或 `/quit` | 退出 Claude Code |
| `/cost` | 查看本次会话的 API 使用成本 |
| `/model` | 切换使用的模型 |
| `/agent` | 进入/退出代理模式 |

### 示例

```
> /help          # 查看所有可用命令
> /clear         # 清空当前对话
> /cost          # 查看花费
```

## 最佳实践

### 1. 明确表达需求

**不好的请求：**
```
> 修复这个 bug
```

**好的请求：**
```
> 在 src/utils.js 中，函数 calculateTotal 没有正确处理负数输入，请修复这个问题
```

### 2. 分步骤处理复杂任务

对于大任务，分成多个小步骤：

```
> 第一步：帮我分析当前项目的依赖结构
> 第二步：找出哪些依赖可以更新
> 第三步：执行安全的依赖更新
```

### 3. 利用上下文

Claude 会记住当前会话的上下文，可以引用之前的内容：

```
> 基于刚才的分析，帮我写一个优化方案
```

### 4. 验证重要操作

对于重要的代码修改，先让 Claude 解释变更内容：

```
> 请解释一下这个修改的具体内容
> 确认一下这个操作会影响哪些文件
```

### 5. 结合版本控制

在使用 Claude Code 进行大规模修改前：

```bash
# 先提交当前更改
git add .
git commit -m "准备使用 Claude Code 重构"

# 然后使用 Claude
git diff  # 查看 Claude 做了哪些修改
```

## 常见问题

### Q: Claude Code 和网页版 Claude 有什么区别？

**A:** Claude Code 是专为开发工作设计的 CLI 工具，可以：
- 直接读取和编辑你的代码文件
- 执行 bash 命令
- 搜索代码库
- 更适合软件开发工作流

网页版 Claude 更适合一般性对话和不需要访问本地文件的任务。

### Q: Claude Code 会将我的代码发送到云端吗？

**A:** 是的，Claude Code 需要将你提供的问题和相关代码发送到 Anthropic 的服务器进行处理。建议：
- 不要将敏感凭证（密码、密钥）发送给 Claude
- 遵守你所在公司的数据安全政策
- 注意查看隐私政策了解详情

### Q: 如何提高 Claude Code 的响应质量？

**A:** 
1. **提供上下文**：告诉 Claude 项目背景、技术栈等
2. **具体说明需求**：明确说出你想要什么
3. **分步骤提问**：复杂任务分解成小步骤
4. **给出示例**：如果可能，提供期望输出的例子
5. **迭代优化**：基于回答继续追问和完善

### Q: Claude Code 支持哪些编程语言？

**A:** Claude Code 理论上支持任何编程语言，因为它主要通过文件操作和命令行来工作。Claude AI 本身对以下语言特别熟悉：
- JavaScript / TypeScript
- Python
- Java / Kotlin
- Go
- Rust
- C / C++
- Ruby
- PHP
- Swift

### Q: 使用 Claude Code 会消耗 API 额度吗？

**A:** 是的，Claude Code 使用 Anthropic API，会产生相应费用。可以使用 `/cost` 命令查看当前会话的花费。建议：
- 合理使用，避免无意义的重复请求
- 对于大批量操作，先小范围测试
- 关注 Anthropic 的定价页面了解费率

### Q: 如何退出 Claude Code？

**A:** 有以下几种方式：
- 输入 `/exit` 或 `/quit`
- 按 Ctrl+C（可能需要两次）
- 按 Ctrl+D
